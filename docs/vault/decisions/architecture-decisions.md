## Dark Premium Glassmorphism

Körset — приложение для покупателей в супермаркетах. Сценарий: стоит в магазине, телефон в одной руке, сканирует штрихкод.

Почему тёмный:
1. Витринный эффект — Glassmorphism = премиум. B2B2C: магазины платят за подписку, интерфейс должен выглядеть дорого
2. Тёмная тема в ярком магазине — лучше читается
3. Экономия батареи OLED — тёмные пиксели = меньше энергии
4. Сканер-контекст — тёмная тема = плавный переход к камере
5. Дифференциация — большинство SaaS светлые, Körset выделяется

Технически: Vanilla CSS, `backdrop-filter: blur(20px)`, `rgba(255,255,255,0.05)`, Advent Pro + Manrope.
Проблема: ~1,184+ inline `style={{}}` — нужно вынести в CSS-классы.

## Fit-Check детерминированный (не полностью AI)

Два подхода: полностью AI vs IF/ELSE для критического + AI для сомнительного.

Выбран: **детерминированный для Red/Orange, AI для Yellow.**

Причины:
1. LLM галлюцинации — цена ошибки для аллергий = здоровье
2. Детерминизм = повторяемость — один продукт + профиль = всегда одинаковый результат
3. Скорость — IF/ELSE мгновенно, LLM 1-3с, для сканера критично
4. Стоимость — детерминированные проверки бесплатны
5. ТР ТС 022/2011 — стандарт чётко определяет аллергены и пороги

AI уместен: Yellow level (Халал/Lifestyle), AI-чат (расшифровка состава), RAG-контекст.
AI НЕ делает: Red level, НЕ заменяет вердикт, НЕ работает без детерминированного базиса.

## Supabase pgvector вместо Pinecone

Причины:
1. Нет нового аккаунта/биллинга — Supabase уже подключён
2. Нет новой зависимости — тот же supabase-js
3. Бесплатно — в рамках Supabase free plan (500 MB)
4. Единая база — JOIN между vault_embeddings и products
5. RLS — Row Level Security на векторах
6. Масштаб — для тысяч чанков pgvector + HNSW достаточно

Переключиться на Pinecone когда: 100K+ чанков, latency <10ms, мультирегиональная репликация.

Embedding модель: `text-embedding-3-small`, 1536 dim, $0.02/1M токенов.
