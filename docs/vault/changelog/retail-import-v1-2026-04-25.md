# RetailImport V1 — 2026-04-25

## Что изменилось

- `RetailImportScreen` перестал быть заглушкой и стал рабочим экраном импорта прайс-листа для Retail Cabinet.
- Поддержаны CSV, XLS и XLSX до 5MB.
- Экран показывает предпросмотр валидных строк, количество ошибок, количество неизвестных EAN и отчёт после применения.
- UI-тексты вынесены в общий `useI18n` словарь: `retail.import` для RU/KZ.

## Как работает импорт

1. Пользователь загружает файл с колонками вроде `EAN`, `Цена`, `Наличие`, `Полка`, `Название`.
2. `src/utils/retailImport.js` парсит файл, нормализует заголовки, валидирует EAN и цену, убирает дубли.
3. При применении код ищет товары только внутри текущего `store_id`.
4. Для найденных товаров обновляются `price_kzt`, `stock_status`, `shelf_zone`, `updated_at`.
5. Неизвестные EAN не создаются автоматически и возвращаются в отчёт.

## Почему так

Автоматическое создание новых `global_products` из прайс-листа опасно для Data Moat: магазины могут прислать неполные названия, ошибочные EAN, мусорные строки или локальные позиции. Поэтому V1 делает безопасный B2B-шаг: быстро обновляет цены/наличие для существующего ассортимента и отдельно показывает неизвестные EAN как материал для data-quality pipeline.
