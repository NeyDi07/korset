# Архитектура и редизайн ProfileScreen (Roadmap)

> Домен: plans
> Дата: 2026-05-03
> Статус: Утверждено к разработке, разделено на этапы

## Текущая проблема
Текущий `ProfileScreen` содержит хаотично разбросанные пункты меню. Разделы «Помощь» и «Обратная связь» не имеют обработчиков кликов (мертвые кнопки). Отсутствует важная юридическая база для снятия медицинской ответственности (Terms of Service).

## Идеальная структура UI (Утверждено)

Структура пересобрана по стандартам B2C продуктов (Kaspi, Wolt, Apple) в 4 основных блока + 1 блок для владельцев бизнеса.

### Блок 1: Аккаунт
- **Личные данные** (`/account`) — управление email, смена пароля, удаление аккаунта, ID пользователя.
- **Приватность** (`/privacy`) — управление хранением истории (локально или облако).

### Блок 2: Настройки
- **Уведомления** (`/notifications`) — настройка пушей (перенесено из Основного блока).
- **Язык** — переключатель RU / KZ (inline).
- **Оформление** — переключатель Светлая / Темная тема (inline).
- **Звук и вибрация** (`/sound-settings`) — новые переключатели для настройки сканера (писк и haptic).
- **Перезапустить обучение** — сброс флага онбординга (перенесено из футера).

### Блок 3: Поддержка
- **Вопросы и ответы (FAQ)** (`/faq`) — как работает сканер, расшифровка цветов (Красный/Оранжевый/Желтый), халал-база.
- **Служба поддержки** — кнопка вызывает `SupportBottomSheet` с переходом в Telegram бота (WhatsApp исключен для V1).

### Блок 4: О приложении (Юридический блок)
- **О Körset** (`/about`) — версия, логотип, ссылки на Instagram, миссия "Сделано в Казахстане".
- **Пользовательское соглашение (Terms of Service)** (`/terms`) — **Критический документ**. Описывает отказ от медицинской ответственности, указывает, что Körset — это информационный сервис, и ответственность за проверку упаковки лежит на пользователе (Закон РК).
- **Политика конфиденциальности** (`/privacy-policy`) — правила сбора email и истории.

### Блок 5: Для партнеров (только для owners)
- **Управление магазином** (`/retail/:slug`) — доступ в дашборд B2B.

---

## Roadmap по сессиям (Этапы)

Каждый этап должен выполняться в отдельной AI-сессии для максимального качества.

- **Этап 1: Архитектура UI профиля** — Переписывание `ProfileScreen.jsx`. Создание пустых файлов для новых экранов (`AboutScreen`, `TermsScreen`, `SoundSettingsScreen`, `FaqScreen`) и настройка роутинга в `routes.js`.
- **Этап 2: Блок "Настройки"** — Реализация экрана `/sound-settings` и хука `useSettings` (localStorage) для управления звуком/вибрацией сканера.
- **Этап 3: Блок "Поддержка"** — Реализация компонента `SupportBottomSheet` с UI-эффектами и кнопкой Telegram (deep link).
- **Этап 4: Блок "Юридический и О приложении"** — Верстка экрана `/about`. Написание грамотного юридического текста для `/terms` (Отказ от медицинской ответственности по законам РК).
- **Этап 5: Блок "FAQ / Как это работает"** — Написание качественного текста про Fit-Check, ИИ и базы NPC. Создание экрана с аккордеонами (открывающиеся списки).
