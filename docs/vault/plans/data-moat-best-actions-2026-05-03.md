# 2026-05-03 — Data Moat best actions

Цель: зафиксировать лучшее применение Codex для следующего крупного направления Körset — Data Moat как продуктовый и бизнес-актив, а не просто набор парсеров.

## Главный вывод

Самый сильный next step — построить Data Trust Layer: единый слой достоверности данных, который управляет источниками, качеством, свежестью, отображением доверия в UI и приоритетом данных для Fit-Check.

## Почему это важнее обычного enrichment

- Data Moat уже частично реализован: `data_quality_score`, `sourceMeta`, TTL для OFF, каскад resolver и AI-маркировка существуют.
- Главный пробел теперь не только в новых источниках, а в том, что пользователь и магазин не видят уровень доверия к данным.
- Для B2B это можно продавать как "проверенный каталог магазина" и "рост покрытия безопасных сканов", а не как техническую базу штрихкодов.

## Рекомендуемая последовательность

1. Data Trust Layer: нормализовать `sourceConfidence`, `qualityScore`, `sourceMeta`, freshness и source badges.
2. ProductScreen/Retail UI: показать доверие к данным, предупреждения для AI/низкого качества и понятные статусы для магазина.
3. Quality Gates: Fit-Check должен менять тональность при `qualityScore < 80`, особенно для аллергенов и халал.
4. Source Registry: единый реестр источников NPC, Arbuz, USDA, OFF, AI, store_manual с весами, TTL и разрешением конфликтов.
5. Retail Data Ops: экран/виджет для владельца магазина: сколько товаров подтверждено, сколько без состава, сколько требует проверки.
6. Advanced Moat: партнерства/интеграции с GS1 KZ, NPC, Halal Damu, EAN-DB, CommerceML.

## Вопросы к владельцу

- Что для первых продаж важнее: доверие покупателя в ProductScreen или аргумент для владельца магазина в Retail Cabinet?
- Готов ли владелец показывать пользователю предупреждения "данные предварительные", даже если это снижает магию AI?
- Нужно ли в V1 строить официальный путь через GS1/Halal Damu или сначала усилить текущий каталог и импорт магазинов?
- Будет ли магазин сам подтверждать состав/халал/цены, или Körset должен брать на себя роль верификатора?
