# Name Normalization System

> Этап 3 data enrichment pipeline. Применён 2026-05-02.

## Проблема

7008 продуктов в БД имели грязные названия:
- 934 ALL CAPS (складская этикетка)
- 560 мусорных суффиксов упаковки (ПЭТ, Ж/Б, КНВРТ...)
- 815 некрасивого форматирования % (2,5% → 2,5 %)
- 1558 некрасивого форматирования веса (500ГР → 500 г)
- 295 двойного веса
- 235 длинных имён >80 символов

## Решение: `normalizeName()`

**Файл:** `src/domain/product/nameNormalizer.js`

### Алгоритм (7 шагов, строго по порядку)

1. **Удалить аббревиатуры упаковки** (14+ паттернов): КНВРТ, ТБА, Т/Б, Ж/Б, ЖБ, П/Б, ПБ, ПЭТ, ПЕТ, П/Э, ТБ, С/Б, СТ/Б, СТБ, СТ.Б
2. **Удалить словесные суффиксы упаковки** в конце имени: стекло, пакет, дой-пак, тетрапак, жесть и т.д.
3. **Дедупликация двойного веса** — если вес появляется 2+ раза с одинаковым числом → удалить второе
4. **Sentence case** (если имя было ALL CAPS):
   - lowercase всё
   - Первое слово → uppercase
   - Латинские слова (2+ буквы) → Title Case (бренды)
   - После цифры+пробел → uppercase
   - Параметр `brand` → восстановить кейс бренда
5. **Форматировать жирность %**: `2,5%` → `2,5 %`, `3.2%` → `3,2 %`
6. **Форматировать вес/объём**: `500ГР` → `500 г`, `1Л` → `1 л`, точка→запятая
7. **Обрезка** >80 символов по границе слова

### Критичный порядок в pipeline

```
1. Сырые данные из API
2. extractAllAttributes({ name }) — ИЗ СЫРОГО имени
3. normalizeCategory() — ПО СЫРОМУ имени
4. normalizeName(name, { brand }) — ПОСЛЕ извлечения
5. Сохранить в БД
```

**Почему:** attributeExtractor и classifyByName работают с сырым именем (ищут ПЭТ, Ж/Б, %, etc.). Если сначала нормализовать — атрибуты теряются.

## Принятые решения

1. **Бренд остаётся в name** — удалять = непонятно какой продукт. Поиск по бренду уже работает.
2. **Жирность и вес остаются в name** — потребителю нужно видеть это сразу. Форматируются красиво.
3. **Удаляется ТОЛЬКО мусор** — суффиксы упаковки (ПЭТ, Ж/Б), двойной вес, NPC-мусор.
4. **Sentence case, не Title Case** — проще, меньше ошибок с предлогами/союзами.
5. **missing_products.local_name НЕ нормализуется** — ценен как оригинал от магазина.
6. **Краевые случаи** — "П\ПАКЕТ" (backslash), mixed-case мусор — не обрабатываются (приемлемо).
