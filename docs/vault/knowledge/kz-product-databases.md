## 1. BARCODE / PRODUCT DATABASES

### 1.1 GS1 Kazakhstan (ОФИЦИАЛЬНЫЙ) ⭐⭐⭐⭐⭐

- **URL:** https://gs1.kz
- **Что даёт:** Префикс «487» для Казахстана. Реестр всех KZ штрихкодов. GEPIR-поиск (проверить штрих-код). DB вход: https://db.gs1.kz
- **API:** ✅ Служба API проверки GTIN — POST HTTPS, JSON in/out. Проверяет: целостность GTIN, валидность GCP, наличие данных в Глобальном реестре продуктов GS1 (200M+ товаров)
- **Доступ:** Только для членов GS1 KZ с API-ключом. Контакт: d.seitkanov@gs1.kz
- **Стоимость:** Членство в GS1 (вступительный взнос + ежегодные взносы) + API-ключ по запросу
- **Покрытие KZ:** ВСЕ казахстанские штрихкоды (487-префикс) + глобальный реестр 200M+
- **Данные:** GTIN, GLN, GPC категории, GDSN синхронизация, GS1 Cloud
- **Вердикт:** MUST-HAVE. Это единственный официальный источник для верификации KZ штрихкодов. Нужно вступить в GS1 и получить API-ключ.

### 1.2 National Product Catalogue (NPC) — НОВОЕ 2026! ⭐⭐⭐⭐⭐

- **URL:** Нет публичного (правительственная система)
- **Источник:** https://www.roedl.com/en/insights/national-product-catalogue-kazakhstan/
- **Что даёт:** Единая государственная цифровая БД всех товаров на рынке KZ. Каждый товар получает уникальный идентификационный код.
- **Интеграции:** Электронные счета-фактуры НДС, кассовые системы, система маркировки
- **Обязательные данные при регистрации:**
  - Характеристики и параметры товара (размеры, вес, состав)
  - Информация о стандартах/технической документации
  - Документы подтверждения соответствия
- **Статус:** Пилотный режим с января 2026, полный запуск — июль 2026
- **Стоимость:** Государственная система (вероятно бесплатная для участников рынка)
- **Покрытие KZ:** ПОЛНОЕ (обязательная регистрация для всех товаров)
- **Вердикт:** КРИТИЧЕСКИ ВАЖНО. Когда запустится полностью — это будет единый источник правды для ВСЕХ товаров KZ. Нужно мониторить и подключаться.

### 1.3 EAN-DB ⭐⭐⭐⭐

- **URL:** https://ean-db.com
- **Что даёт:** 69M+ товаров по штрихкоду. API с детальной информацией.
- **Данные по API:**
  - Штрихкод (EAN-8/13/14, UPC, ISBN)
  - Название (многоязычное, включая русский — 13.9M на русском!)
  - Категории (Google product taxonomy)
  - Производитель (+ wikidata ID)
  - Изображения (39% товаров)
  - **metadata.food** — КБЖУ на 100г (energy, proteins, fat, carbs, sugars, fiber, sodium, etc.) — 1.3M товаров
  - **metadata.food.nutriScore** — 790K товаров
  - **metadata.generic.ingredients** — Состав с разбором (E-добавки, vegan/vegetarian, под-ингредиенты) — по ингредиентам
  - Страна штрихкода
- **API:** ✅ REST API, JWT авторизация, Swagger playground
  - `GET /api/v2/product/{barcode}`
  - Mass dump: 0.005 EUR за штрих-код
- **Стоимость:** 250 запросов бесплатно, далее:
  - 5K за 9 EUR
  - 50K за 69 EUR
  - 300K за 249 EUR
  - Дампы: 0.005 EUR/штрихкод
- **Покрытие KZ:** 487-префикс есть, но покрытие KZ-специфичных товаров неизвестно
- **Вердикт:** BEST OPTION для старта. Русские названия + ингредиенты + КБЖУ + NutriScore. Можно купить дамп по 487-префиксу.

### 1.4 Open Food Facts ⭐⭐⭐

- **URL:** https://world.openfoodfacts.org
- **Что даёт:** Свободная БД пищевых продуктов с фотографиями, составом, КБЖУ
- **API:** ✅ Бесплатный, REST API
  - `GET /api/v2/product/{barcode}`
  - `GET /api/v2/search?countries=Kazakhstan`
- **Данные:** Название, бренд, состав, КБЖУ, аллергены, категории, фото, NutriScore, Nova group
- **Стоимость:** Полностью бесплатная (open source)
- **Покрытие KZ:** Ограниченное (добровольные добавления)
- **Вердикт:** Бесплатный fallback. Можно стимулировать добавление KZ товаров.

### 1.5 Barcode-list.ru ⭐⭐

- **URL:** https://barcode-list.ru
- **Что даёт:** 5M+ штрихкодов с наименованиями товаров
- **Данные:** Штрихкод + название (статистически наиболее употребимое)
- **API:** ❌ Только веб-поиск
- **Стоимость:** Бесплатно (для Microinvest Склад Pro встроена)
- **Покрытие KZ:** Россия-ориентированная, но часть товаров общая для ЕАЭС
- **Вердикт:** Полезна для начального наполнения. Ограниченная ценность (только штрихкод + название).

### 1.6 GitHub: open barcodes (aioke/barcodes) ⭐⭐

- **URL:** https://github.com/aioke/barcodes
- **Что даёт:** 1.8M штрихкодов, Россия 2021-2022, CSV + SQLite
- **Данные:** Id, Category, Vendor, Name, Article, Barcode
- **API:** ❌ Скачать файл
- **Стоимость:** Бесплатно (CC0)
- **Покрытие KZ:** Косвенное (РФ-рынок, но ЕАЭС-товары пересекаются)
- **Вердикт:** Разовая загрузка для начальной базы.

### 1.7 KazDATA ⭐

- **URL:** https://kazdata.kz/30/kazdata-API.html
- **Что даёт:** БД организаций Казахстана (БИН, ОКПО, наименование, ОКЭД, КАТО, КРП)
- **API:** ✅ JSON API с авторизацией (2-этапная: salt + hash)
- **Стоимость:** Платная (по запросу менеджеру)
- **Покрытие KZ:** Полное по организациям, но НЕТ данных по товарам
- **Вердикт:** Не подходит для товаров. Полезно только для данных о магазинах/производителях.

### 1.8 POSCENTER.kz ⭐

- **URL:** https://poscenter.kz/bazy-shtrikh-kodov/
- **Что даёт:** Базы штрихкодов для загрузки в кассовые системы (1С, UMAG)
- **Стоимость:** Требуется регистрация
- **Покрытие KZ:** Казахстан-ориентированная
- **Вердикт:** Возможно полезна, но требует регистрации для оценки.

- **API:** ❌ Нет публичного API (только веб)
- **Покрытие KZ:** Универсальные продукты (не бренд-специфичные)
- **Вердикт:** Полезно для generic продуктов (яблоко, говядина), но не для брендовых товаров по штрихкоду.

## 3. HALAL CERTIFICATION

### 3.1 Халал Даму (ДУМК) — ОФИЦИАЛЬНЫЙ ⭐⭐⭐⭐⭐

- **URL:** https://halaldamu.kz (портал) + https://halalinfo.kz (инфо-сайт)
- **Что даёт:** Реестр сертифицированных предприятий, проверка по названию/категории
- **Категории реестра:** Мясо, молочка, кондитерские, консервы, крупы, масло, напитки, полуфабрикаты, снэки, детское питание, общепит, косметика и др. (40+ категорий)
- **Статистика:** 2500+ сертифицированных предприятий, 50+ профессионалов, 12 лет на рынке
- **Инструменты:**
  - Веб-реестр: https://halaldamu.kz/reestr/ (поиск по названию + категория)
  - Telegram-бот @halaldamukz
  - Мобильное приложение «Halal Guide KZ» (iOS + Android) — сканирует ингредиенты
  - QR-код на сертификатах
  - Логотип «QMDB Halal» на упаковке
- **API:** ❌ Нет публичного API (только веб-реестр + бот)
- **Аккредитация:** №KZ.H.01.0002 в Национальной системе аккредитации РК (СТ РК OIC/SMIIC 2-2022)
- **Международное признание:** JAKIM (Малайзия), SMIIC, BPJPH (Индонезия)
- **Стоимость:** Сертификат выдаётся на 1 год, стоимость по запросу
- **Вердикт:** MUST-HAVE. Единственный официальный орган при ДУМК. Реестр можно парсить + Telegram-бот для проверки. Нужно наладить партнёрство для API-доступа.

### 3.2 Ассоциация Халал Индустрии Казахстана (AHIK) ⭐⭐⭐⭐

- **URL:** https://halal-kz.kz
- **Что даёт:** Международная сертификация халал, 700+ отечественных компаний, 50+ международных
- **Поиск предприятий:** https://halal-kz.kz/ru/search
- **Признание:** JAKIM (Малайзия), International Halal Integrity Alliance
- **API:** ❌ Нет публичного API
- **Вердикт:** Второй по значимости сертификатор. Реестр тоже нужно парсить.

### 3.3 КазСтандарт (KazStandard) ⭐⭐⭐

- **URL:** https://ksm.kz/ru/halal/certification-process/
- **Что даёт:** Халал-сертификация через государственный институт стандартизации
- **Сертификат:** QR-код для проверки подлинности, на 3 языка (каз/рус/англ)
- **API:** ❌ Нет публичного API
- **Вердикт:** Третий сертификатор, государственный. Нужно учитывать все три органа.

## 4. KASPI.KZ PRODUCT DATA

### 4.1 Kaspi Магазин ⭐⭐

- **URL:** https://kaspi.kz/shop/
- **Что даёт:** Крупнейший маркетплейс KZ с каталогом товаров, ценами, отзывами, характеристиками
- **Структура данных товара:** Название, бренд, цена, характеристики, отзывы, рейтинг, фото, категория
- **Merchant API:** Существует для продавцов (Kaspi Merchant), но:
  - Доступ только для зарегистрированных продавцов
  - Нет публичной документации
  - Интеграция через Kaspi Partner API (для управления каталогом продавца)
- **Парсинг:** Теоретически возможно, но:
  - Защита от ботов
  - Товары по `/shop/p/{product_id}/`, не по штрихкодам
  - Нет прямого соответствия штрихкод → товар Kaspi
- **GitHub:** Нет найденных проектов для работы с Kaspi product data
- **Вердикт:** Ограниченная ценность напрямую. Данные продавцов через Merchant API доступны только партнёрам. Парсинг сложен и юридически рискован.

## 5. 1C / COMMERCXML ИНТЕГРАЦИЯ

### 5.1 CommerceML ⭐⭐⭐⭐

- **Формат:** Стандартный XML-формат обмена каталогами товаров в 1С
- **Использование:** Де-факто стандарт в KZ ритейле
- **Данные из 1С:** Номенклатура, штрихкоды, цены, остатки, характеристики, единицы измерения
- **Экспорт из 1С:** 1С:Розница, 1С:Управление торговлей, 1С:Комплексная автоматизация
- **Для Körset:** При интеграции с магазином через 1С → автоматический импорт его номенклатуры со штрихкодами
- **Вердикт:** КЛЮЧЕВОЙ канал получения данных от магазинов. Поставщики магазинов отдают прайс-листы в CommerceML.

### 5.2 МойСклад API ⭐⭐⭐

- **URL:** https://www.moysklad.ru/api/
- **Что даёт:** REST API для управления каталогом, штрихкодами, ценами, остатками
- **Популярность:** Используется малым бизнесом в KZ
- **API:** ✅ Полный REST API с JSON
- **Вердикт:** Альтернатива 1С для малого бизнеса. Меньше распространён в KZ.

### 5.3 СБИС ⭐⭐

- **URL:** https://sbis.ru
- **Что даёт:** ЭДО + каталог товаров + аналитика
- **API:** ✅ Есть API для интеграции
- **Вердикт:** Более ориентирован на документооборот, чем на каталог товаров.

## 6. ПРАВИТЕЛЬСТВЕННЫЕ И ОТКРЫТЫЕ БАЗЫ KZ

### 6.1 Казахстанское содержание (ks.gov.kz) ⭐⭐⭐

- **URL:** https://ks.gov.kz
- **Что даёт:**
  - База данных ТРУ (товары, работы, услуги) и их поставщиков
  - Единый каталог товаров
  - Сертификаты «СТ-KZ» (казахстанское содержание)
  - Реестр отечественных производителей
- **API:** ❌ Веб-интерфейс только
- **Стоимость:** Бесплатно
- **Покрытие KZ:** Казахстанские производители
- **Вердикт:** Полезно для определения KZ-производителей и СТ-KZ сертификатов.

### 6.2 data.egov.kz (Открытые данные) ⭐⭐

- **URL:** https://data.egov.kz
- **API:** ✅ Elasticsearch API v4, JSON, HTTPS, бесплатно
- **Данные:** Различные государственные датасеты (мед. организации, фармации, преступность и т.д.)
- **Покрытие KZ:** Нет прямого датасета по штрихкодам товаров, но может появиться с NPC
- **Вердикт:** Мониторить. С введением NPC могут добавить датасет по товарам.

## 8. РЕКОМЕНДАЦИИ

### Немедленно (P0):
1. **Зарегистрироваться в GS1 Kazakhstan** — получить доступ к GTIN API и db.gs1.kz
2. **Подключить EAN-DB API** — купить пакет 50K запросов (69 EUR), протестировать покрытие 487-префикса
3. **Начать парсинг Халал Даму реестра** — 2500+ предприятий, 40+ категорий

### Среднесрочно (P1):
4. **Мониторить NPC запуск** — к июлю 2026 может стать главным источником данных
5. **Настроить Open Food Facts** как бесплатный fallback для состава/КБЖУ
6. **Разработать CommerceML импорт** — для получения каталогов от магазинов

7. **Партнёрство с Халал Даму** — получить API-доступ к реестру
8. **Партнёрство с GS1 KZ** — полноценный GDSN доступ
9. **Стимулировать Open Food Facts** добавление KZ товаров через бота
