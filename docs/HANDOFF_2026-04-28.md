**6.1. `data_quality_score` column + computation trigger**

```sql
-- Migration 022 (вы создаёте)
ALTER TABLE global_products
  ADD COLUMN data_quality_score smallint
  CHECK (data_quality_score BETWEEN 0 AND 100);

CREATE OR REPLACE FUNCTION calculate_data_quality()
RETURNS trigger AS $$
DECLARE
  score smallint := 0;
BEGIN
  -- 30 points: основные поля
  IF NEW.name IS NOT NULL AND length(NEW.name) > 3 THEN score := score + 10; END IF;
  IF NEW.brand IS NOT NULL THEN score := score + 5; END IF;
  IF NEW.image_url IS NOT NULL THEN score := score + 10; END IF;
  IF NEW.category IS NOT NULL THEN score := score + 5; END IF;

  -- 30 points: состав + питательная ценность
  IF NEW.ingredients IS NOT NULL AND length(NEW.ingredients) > 20 THEN score := score + 15; END IF;
  IF NEW.nutriments_json IS NOT NULL AND jsonb_typeof(NEW.nutriments_json) = 'object' THEN score := score + 15; END IF;

  -- 20 points: аллергены + диета
  IF jsonb_array_length(COALESCE(NEW.allergens_json, '[]'::jsonb)) > 0 THEN score := score + 10; END IF;
  IF jsonb_array_length(COALESCE(NEW.diet_tags_json, '[]'::jsonb)) > 0 THEN score := score + 10; END IF;

  -- 20 points: KZ-specific
  IF NEW.name_kz IS NOT NULL THEN score := score + 10; END IF;
  IF NEW.halal_status = 'halal_certified' THEN score := score + 10; END IF;

  NEW.data_quality_score := score;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER set_data_quality
  BEFORE INSERT OR UPDATE ON global_products
  FOR EACH ROW EXECUTE FUNCTION calculate_data_quality();
```

### Git workflow

1. **Не делай гигантских коммитов.** Разбивай работу на 3-5 структурированных коммитов в одной сессии.
2. **Conventional commits**:
   - `feat(scope): ...` — новая функциональность
   - `fix(scope): ...` — фикс
   - `refactor(scope): ...` — рефакторинг без изменения поведения
   - `docs: ...` — только документация
3. **Не используй `-m "msg" -m "msg"` в PowerShell** — он коряво парсит. Пиши длинный message в файл `.git/COMMIT_MSG_TMP.txt` и коммить через `git commit -F .git/COMMIT_MSG_TMP.txt`.
4. **Husky pre-commit запускает lint-staged** (~30 сек на коммит). Это нормально, не используй `--no-verify`.
5. **Push автодеплоит Vercel** (NeyDi07 GitHub аккаунт). НЕ запускай `vercel --prod` вручную.
6. **Каждый коммит должен пройти**: `npm run test:unit` + `npm run build` + `npm run lint`.
