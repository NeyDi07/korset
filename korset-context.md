# KÖRSET — Файл передачи контекста (Handover Document)

Этот файл содержит исчерпывающую информацию о проекте Körset, его целях, текущем состоянии, технической архитектуре и планах на будущее. Предназначен для бесшовной передачи проекта другому ИИ-ассистенту.

---

## 1. Концепция и Миссия (Vision)
**Körset** — это высокотехнологичное mobile-first PWA / веб-приложение, созданное специально для офлайн-ритейла в Казахстане (KZ). 

### Основная идея:
Покупатель в магазине сканирует штрихкод товара и мгновенно получает:
- **Fit-Check:** Подходит ли товар именно ему (на основе здоровья и религии).
- **Халал статус:** Проверка на соответствие стандартам.
- **Детектор аллергенов:** Мгновенное предупреждение об опасности.
- **Расшифровка состава:** Обычный текст состава превращается в понятные категории (добавки, сахар, консерванты).
- **AI-Вердикт:** Краткое резюме от нейросети о качестве продукта.
- **Альтернативы:** Предложение более здоровых вариантов, если текущий не подходит.

---

## 2. Бизнес-модель и Цели
Körset — это не просто приложение для покупателей, это **B2B SaaS-продукт** для ритейлеров.

- **Целевая аудитория:** Крупные торговые сети Казахстана (Magnum, ATB и др.).
- **Монетизация:** Продажа подписки магазинам. Магазины получают вовлеченность клиентов, аналитику предпочтений и современный имидж.
- **Главная цель:** Создать реальный, стабильный продукт "product-level" качества, готовый к запуску в торговых залах.

---

## 3. Техническая архитектура
- **Frontend Stack:** React 18, Vite, JavaScript (ESM).
- **UI/UX Дизайн:**
  - Стиль: **Dark Premium Glassmorphism** (эффект стекла, глубокие темные тона).
  - Шрифты: `Advent Pro` (заголовки), `Manrope` (основной текст).
  - Иконки: **Material Symbols Outlined** (подключены через Google Fonts).
- **Backend:** Supabase (Auth, PostgreSQL DB, Storage для аватаров).
- **State Management:** React Context API (Profile, Auth, Store, UserData).
- **Routing:** `react-router-dom` v6. Архитектура роутов завязана на `:storeSlug` (например, `/s/atb/catalog`).

## 4. Что уже реализовано (Текущий статус)

### Лендинг (Landing Page):
- Премиальный дизайн с вертикальным стеком функций.
- Работающие кнопки «Выбрать магазин» и «О проекте» (плавный скролл к блоку `b2b-section`).
- Исправленные иконки (Center Focus, Psychology, Fact Check и др.).

### Онбординг и Профиль:
- 2-шаговый Onboarding (Имя -> Аватар).
- **Edit Mode:** Специальный режим редактирования (`/setup-profile?mode=edit`) для быстрой смены имени и фото без сброса настроек диеты.
- **Система аватаров:** Компонент `ProfileAvatar` корректно отображает как AI-пресеты (на CSS), так и загруженные пользователем фото из Supabase Storage.

### Скан и Продукты:
- **Scan Flow:** Работающий сканер штрихкодов.
- **AI Screen:** Интерактивный чат с ИИ (Körset AI), который знает контекст конкретного продукта.
- **Поддержка языков (RU/KZ):** Глобальная система переводов через контекст `useI18n`.

## 5. Дорожная карта (Roadmap) и Будущие идеи

### Приоритет 1: Юридическая чистота
- **Юридический Дисклеймер:** Обязательная плашка (Disclaimer), предупреждающая об ошибках ИИ и аллергических рисках. Работа с Privacy Policy и согласными на обработку данных.

### Приоритет 2: Киллер-фичи для пользователей
- **Актуальные цены:** Отображение цен из базы магазина (так как ценники в залах часто висят неправильно).
- **Трекер сроков годности:** Раздел для сохранения купленных товаров с напоминаниями о порче.
- **Карта магазинов:** Интерактивная карта торговых точек, поддерживающих Körset.
- **Stories / Советы дня:** Формат "Stories" на главном экране (лайфхаки, рецепты, новости).
- **История сканов:** Добавление графиков потребления, аналитики сахара/калорий за неделю.

### Приоритет 3: B2B Функционал
- **Admin Dashboard:** Интерфейс для администрации магазинов с аналитикой: что сканируют, что ищут, какие аллергены преобладают в данном районе.

---

## 6. Важные идеи от Пользователя (Mental Model)
- **Цифровая память питания:** Приложение должно хранить историю не просто как список, а как "базу данных о здоровье" пользователя.
- **Fit-Check:** Это мгновенное сравнение состава продукта с "цифровым профилем здоровья" пользователя.
- **Локальный контекст:** Опора на штрихкоды и продукты, типичные именно для Казахстана (молочка ФудМастер, Magnum Selection и т.д.).

---

## 7. Инструкции по разработке
- **Стиль:** Сохранять "Dark Premium" и mobile-first. Не делать редизайн без запроса.
- **Иконки:** Использовать только зарезервированные Material Symbols (например, `barcode_scanner`, `psychology`).
- **Маршруты:** Всегда учитывать `storeSlug`. Внутренние ссылки должны формироваться через хелперы в `src/utils/routes.js`.
- **API Ключи:** Настраиваются в `.env` файле.
