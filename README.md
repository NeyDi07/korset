# Körset MVP

AI-ассистент для офлайн-ритейла. Помогает выбрать товар у полки.

## Запуск

```bash
npm install
npm run dev
```

Откроется на http://localhost:5173

## Деплой на Vercel

1. Загрузи проект на GitHub
2. Подключи репозиторий в vercel.com
3. Framework: Vite → Deploy

## Структура

```
src/
├── screens/
│   ├── ProfileScreen.jsx   # Выбор профиля + пресеты
│   ├── ScanScreen.jsx      # Список демо-товаров + кнопка скана
│   ├── ProductScreen.jsx   # Карточка товара + статус ✅/❌
│   ├── AlternativesScreen.jsx  # Альтернативы
│   └── AIScreen.jsx        # AI-помощник (mock)
├── utils/
│   ├── profile.js          # Пресеты и localStorage
│   ├── fitCheck.js         # Логика подходит/не подходит
│   └── mockAI.js           # Mock ответы AI
└── data/
    └── products.json       # 15 демо-товаров
```

## Демо-сценарии (для презентации)

1. **Халал + без молока** → Открыть "Йогурт Простоквашино" → ❌ Не подходит (не халал, содержит молоко) → Альтернативы → CоcoFresh кокосовый ✅

2. **Без сахара** → Открыть "Coca-Cola классическая" → ❌ Содержит сахар → Альтернативы → Cola Zero ✅

3. **Экономия** → Открыть "Кабель TurboLink 100W" → ✅ Подходит → AI: "Почему лучше?"

## Ответы на Q&A жюри

- **Почему QR?** → Стабильность демо. На проде — EAN штрихкод тот же идентификатор.
- **Откуда данные?** → Каталог магазина (Excel) → products.json.
- **Халал подтверждаете?** → Информируем по маркировке производителя, не сертифицируем.
- **Монетизация?** → B2B подписка магазинов. Платит магазин за конверсию и аналитику.
