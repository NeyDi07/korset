## 1. СУТЬ И ФИЛОСОФИЯ ПРОЕКТА (VISION)
**Körset** — это высокотехнологичное mobile-first PWA-приложение для офлайн-ритейла в Казахстане (KZ). 
Его цель — переосмыслить опыт покупок в супермаркетах, став персональным «защитником диеты» и «цифровой памятью питания» для каждого покупателя.

### Ментальная модель (Как это работает для пользователя):
1. Находясь в магазине, покупатель наводит сканер на штрихкод товара.
2. Секундный анализ — и приложение выдает **"Fit-Check"** (вердикт: подходит продукт или нет).
3. Приложение учитывает:
   - **Религиозные фильтры** (строгий Халал).
   - **Пищевые протоколы** (Кето, Веган, Без сахара).
   - **Аллергены** (орехи, лактоза, глютен + кастомные вводы пользователя).
4. Простой текст состава («химия») переводится AI-ассистентом на человеческий язык, выявляя скрытые Е-добавки и консерванты.

### Бизнес-модель (B2B2C):
Несмотря на то, что пользуются приложением обычные люди, **Körset — это B2B-продукт**. 
- Основной клиент: Торговые сети (Magnum, ATB и др.), которые покупают подписку (SaaS-модель).
- Что получают сети: Глубокую аналитику по сканированиям (что ищут люди, какие аллергии преобладают в их районах), повышение лояльности и современный имидж.
- B2C (пользовательский) функционал первичен для создания аудитории, но все решения должны работать на B2B ценность.

## 2. ТЕХНИЧЕСКИЙ СТЕК И АРХИТЕКТУРА
Проект находится на стадии перехода от MVP к стабильному Production-Ready приложению.

- **Frontend:** React 18, Vite, JavaScript (ESM). Архитектура — Single Page Application (SPA).
- **Backend (BaaS):** Supabase (PostgreSQL для базы, Auth для управления пользователями, Storage для загрузки картинок).
- **Стилизация:** Индивидуальный дизайн "Dark Premium Glassmorphism" (эффект стекла, темные тона). В основном Vanilla CSS с элементами Tailwind (через CDN).
- **Иконки:** Исключительно `Material Symbols Outlined` (шрифт подключен через Google Fonts в `index.html`).
- **Роутинг:** `react-router-dom` v6. 
  - **Ключевая фишка роутинга:** У приложения есть "глобальные" экраны (Лендинг `/`, список магазинов `/stores`, Профиль) и "контекстные экраны магазина" (`/s/:storeSlug/catalog`, `/s/:storeSlug/scan`). Приложение динамически подстраивает UI и товары под выбранную сеть.

## 3. ЧТО УЖЕ РЕАЛИЗОВАНО И РАБОТАЕТ (ТЕКУЩЕЕ СОСТОЯНИЕ)

### Интерфейс и Навигация:
- **Landing Page:** Полностью оформлен в стиле Glassmorphism. Вертикально выстроены блоки «особенностей». Кнопки «Выбрать магазин» (переход к списку) и «О проекте» (плавный скролл к якорю `id="b2b-section"`) работают корректно. Иконки заменены на Material Symbols. Не нужные нижние панели навигации (BottomNav) на лендинге скрыты.

### Аутентификация и Профиль:
- Вход/регистрация через Supabase Auth.
- **Онбординг (Setup Profile):** 2-шаговый мастер создания профиля (Шаг 1: Имя, Шаг 2: Аватар).
- **Умное Редактирование (Edit Mode):** Вызов профиля по ссылке `?mode=edit` переключает онбординг в компактный режим (только смена имени и фото на одной странице, без сброса аллергенов).
- **Система аватарок:** Единый компонент `ProfileAvatar.jsx`, который умеет отображать:
  1. Загруженные фото из галереи (URL из Supabase Storage).
  2. Встроенные AI-пресеты (рисуются сложным CSS/SVG кодом внутри React, не требуют внешних картинок).
  3. Резервный вариант (первая буква имени на фиолетовом градиенте).

### Сканирование и AI:
- Рабочий модуль камеры через `html5-qrcode`.
- Экран `AIScreen`: Пользовательский чат с Körset AI. Контекст чата жестко привязан к конкретному отсканированному продукту. ИИ отвечает на основе профиля здоровья (аллергий) пользователя. Встроены быстрые "чипы" (зачем это мне, как готовить, с чем сравнить).

## 4. СТРОГИЕ ПРАВИЛА: ЧЕГО ДЕЛАТЬ НЕЛЬЗЯ (ОШИБКИ ПРОШЛОГО)

**Чтобы избежать поломок в будущем, ИИ должен соблюдать следующие заповеди:**
1. **НЕ ДЕЛАЙ МАССОВЫЙ РЕДИЗАЙН:** Никогда не предлагай переписать стили на стандартные светлые SaaS-шаблоны. У Körset уже есть свой уникальный, дорогой "Dark Premium" вид. 
2. **НЕ ЛОМАЙ РОУТИНГ МАГАЗИНА:** Если ты добавляешь новый экран внутри магазина, он должен иметь путь `/s/:storeSlug/название_экрана`. Не выноси магазинную логику в глобальную область.
3. **НЕ ИСПОЛЬЗУЙ СЛОЖНЫЕ SVG ДЛЯ ИКОНОК:** Мы уже прошли через кривые SVG. Используй только: `<span className="material-symbols-outlined">имя_иконки</span>`.
4. **НЕ МЕНЯЙ ПАРАЛЛЕЛЬНО МНОГО ФАЙЛОВ:** Точечные изменения. Сначала анализ, потом редактирование. Если меняешь логику (как изменение режима редактирования профиля), проверяй, где происходит вызов (`navigate`), чтобы не сломать связь экранов.
5. **Не заменяй `ProfileAvatar`:** Везде, где нужно показать аватар пользователя, используй `<ProfileAvatar avatarId={...} />`. Не пиши хардкод для картинок `<img>`.

## 5. ДОРОЖНАЯ КАРТА (ROADMAP: БУДУЩЕЕ И ЦЕЛИ)

Мы определили следующие приоритетные шаги для развития, которые нужно реализовывать в ближайшем будущем:

### Группа 1: Юридическая безопасность и Доверие (Критично)
- **Дисклеймер ИИ и Аллергенов:** Внедрение обязательных уведомлений (с кнопкой "Я понял"), разъясняющих, что ИИ может ошибаться, а приложение носит рекомендательный характер. Решение о покупке, особенно при сильных аллергиях, берет на себя пользователь.

### Группа 2: «Киллер-фичи» для покупок
- **Актуальные Цены:** Синхронизация с базами магазинов, чтобы показывать правильную цену (так как ценники в физических магазинах часто теряются или устаревают).
- **Срок годности (Anti-Waste):** Добавление кнопки «Купил». Приложение запоминает срок годности отсканированного товара и присылает Push-уведомление за 2 дня до его истечения.
- **Фото из Галереи:** В сканере — возможность загрузить фотографию наклейки состава/штрихкода из сохраненных картинок.
- **Интерактивная карта:** Экран с картой города, где отмечены магазины-партнеры сети Körset.

### Группа 3: Вовлеченность и Аналитика (Retention)
- **Körset Stories (Советы дня):** Внедрение формата "Сториз" на главном экране (короткие карточки про мифы о питании, вред сахара, советы).
- **Расширенная история сканов:** Превращение раздела "История" в дашборд здоровья (графики: сколько сахара сканировал за месяц, сколько E-добавок избежал).
- **B2B Dashboard (Админка):** Разработка скрытой части сайта для менеджеров супермаркетов (просмотр аналитики локации).
- **Повтор Онбординга:** Возможность для пользователя заново запустить вступительные слайды-инструкции из настроек.
