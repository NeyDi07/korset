# KÖRSET — README PROJECT

## Что это
Körset — mobile-first приложение / PWA для офлайн-магазинов, которое помогает пользователю быстро понять, подходит ли ему товар.

Основные сценарии:
- сканирование товара
- просмотр карточки товара
- проверка halal / allergens / ingredients / nutrition
- получение AI explanation
- просмотр alternatives
- работа с preferences / profile

---

## Текущий фокус
Сейчас основной фокус — frontend / product UX.

### Главные зоны внимания:
- onboarding
- profile
- scan flow
- product result screen
- AI screen
- bilingual RU/KZ UX
- consistency UI

---

## Общие правила работы с проектом
1. Сначала анализ, потом изменения
2. Не менять лишние файлы
3. Не трогать архитектуру без необходимости
4. Не ломать уже рабочие экраны
5. Все крупные изменения сначала кратко объяснять
6. Любые risky changes делать аккуратно и поэтапно

---

## Как подходить к задачам
Хороший порядок:
1. Найти нужные файлы
2. Понять текущую реализацию
3. Сформулировать минимальный план изменений
4. Внести изменения
5. Проверить, что не сломались другие части интерфейса

---

## Что особенно хрупкое
При работе быть аккуратным с:
- onboarding
- navigation
- runtime imports / icons / components
- i18n
- AI screen input
- profile/preferences logic
- adaptive mobile layout

---

## Что считается хорошим результатом
Хороший результат — это не просто "код меняется", а:
- экран работает
- стиль не ломается
- мобильная верстка не ломается
- навигация не ломается
- UI становится лучше, а не просто другим
- изменения выглядят как улучшение продукта, а не как случайный редизайн

---

## Что делать, если задача большая
Если задача большая:
- не начинать сразу массовый рефакторинг
- сначала предложить план
- разделить на 2–4 понятных шага
- после этого делать поэтапно
